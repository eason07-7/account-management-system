# 帳戶管理系統 PRD

## 1. 專案概述

開發一個基於網頁的帳戶管理系統，提供用戶登入、註冊、查看帳戶資訊等功能，並區分一般用戶與管理員權限。

## 2. 功能需求

### 2.1 用戶認證

#### 2.1.1 登入功能
- 用戶必須使用帳號密碼登入後才能訪問所有頁面
- 登入成功後跳轉至首頁
- 登入失敗顯示錯誤訊息

#### 2.1.2 註冊功能
- 提供新用戶註冊功能
- 必填欄位：
  - 會員帳號（唯一）
  - 會員姓名
  - 帳號密碼
- 註冊成功後自動登入並跳轉至首頁

#### 2.1.3 登出功能
- 首頁左上角顯示登出按鈕
- 點擊登出後清除登入狀態並跳轉至登入介面

### 2.2 首頁功能

#### 2.2.1 頁面布局
- 左上角顯示目前使用的用戶名（帳號）
- 左上角顯示登出按鈕
- 中央區域顯示所有帳戶資訊列表

#### 2.2.2 帳戶資訊顯示
- 一般用戶可查看所有帳戶資訊
- 顯示內容包含 `infer/預設會員資料.xlsx` 中的所有欄位資訊
- 以表格或卡片形式呈現

### 2.3 管理員功能

#### 2.3.1 權限控制
- 系統區分一般用戶與管理員（admin）帳號組
- 管理員帳號組可編輯所有帳戶資料
- 一般用戶僅能查看，無法編輯

#### 2.3.2 資料管理
- 管理員可新增、編輯、刪除帳戶資訊
- 所有變更即時同步至資料庫

## 3. 技術規格

### 3.1 後端服務
- 使用 Supabase 作為後端服務
- 資料庫儲存用戶帳號、密碼、姓名等資訊
- 實作身份驗證與授權機制

### 3.2 前端技術
- 響應式網頁設計
- 支援現代瀏覽器

### 3.3 部署方案
- 使用 GitHub Pages 進行部署
- 連接 GitHub 倉庫作為源碼倉庫
- 支援自訂域名註冊

## 4. 資料結構

### 4.1 用戶資料表
- 會員帳號（主鍵，唯一）
- 會員姓名
- 帳號密碼（加密儲存）
- 用戶角色（一般用戶/管理員）
- 建立時間
- 更新時間

### 4.2 資料來源
- 初始資料來源：`infer/預設會員資料.xlsx`
- 需將 Excel 資料匯入至 Supabase 資料庫

## 5. 使用者介面需求

### 5.1 登入頁面
- 帳號輸入欄位
- 密碼輸入欄位
- 登入按鈕
- 註冊連結

### 5.2 註冊頁面
- 會員帳號輸入欄位
- 會員姓名輸入欄位
- 帳號密碼輸入欄位
- 確認密碼輸入欄位
- 註冊按鈕
- 返回登入連結

### 5.3 首頁
- 左上角：目前用戶名顯示 + 登出按鈕
- 中央：帳戶資訊列表（表格或卡片）
- 管理員額外顯示：編輯、新增、刪除按鈕

## 6. 安全性需求

- 密碼需加密儲存（使用 Supabase 內建加密機制）
- 所有 API 請求需驗證用戶身份
- 權限控制需在後端實作，前端僅做 UI 顯示控制
- 登入狀態需使用安全的 Session 或 Token 機制

## 7. 非功能需求

- 所有文字內容使用繁體中文
- 介面設計簡潔易用
- 載入速度需流暢
- 支援響應式設計，適配不同裝置

## 8. 開發階段

1. 環境設定：建立 Supabase 專案與資料庫
2. 資料匯入：將 Excel 資料匯入 Supabase
3. 前端開發：實作登入、註冊、首頁功能
4. 權限實作：區分一般用戶與管理員權限
5. 測試：功能測試與安全性測試
6. 部署：設定 GitHub Pages 與域名
