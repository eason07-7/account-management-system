# Cursor 規則說明文件

## 📋 規則總覽

本專案設定了三個核心規則文件，這些規則會自動套用到所有與 AI 的對話中，確保開發過程的一致性和品質。

---

## 1. fundations.mdc（專案基礎規則）

### 🎯 用處（大白話）
這個規則就像是專案的「基本法」，告訴 AI 在寫程式碼時要遵守哪些基本原則。它涵蓋了：
- **程式碼品質**：怎麼寫出乾淨、好讀的程式碼
- **錯誤處理**：當程式出錯時該怎麼處理，不能讓錯誤直接崩潰
- **安全性**：不能把密碼、API 金鑰這些敏感資訊寫在程式碼裡
- **效能**：程式要跑得快，不能讓使用者等太久
- **測試**：重要的功能要寫測試，確保不會壞掉
- **文件**：複雜的地方要寫註解，讓後來的人看得懂

### 💡 點評
**優點：**
- 非常全面，涵蓋了開發的基本面
- 強調安全性，這對帳戶管理系統特別重要
- 注重程式碼品質，有助於長期維護

**建議：**
- 這些規則比較通用，適合所有類型的專案
- 對於你的帳戶管理系統專案來說，安全性規則特別重要，因為涉及使用者資料

---

## 2. front-developer.mdc（前端開發規則）

### 🎯 用處（大白話）
這個規則專門針對前端開發，告訴 AI 在寫網頁時要注意什麼：
- **HTML 結構**：要用對的標籤，不能亂用，要讓網頁結構清楚
- **CSS 樣式**：要讓網站在手機、平板、電腦上都能正常顯示（響應式設計）
- **JavaScript**：要用現代語法，處理非同步操作要正確
- **使用者體驗**：載入時要有提示，錯誤要有訊息，不能讓使用者不知道發生什麼事
- **效能**：圖片要優化，程式碼要分割，不能一次載入太多東西
- **表單處理**：註冊和登入表單要有驗證，不能讓使用者亂填

### 💡 點評
**優點：**
- 非常適合你的專案，因為你的帳戶管理系統就是前端專案
- 強調響應式設計，這對現代網頁很重要
- 注重使用者體驗，這會直接影響使用者對系統的感受

**特別適用於你的專案：**
- 表單處理規則對你的登入、註冊功能特別有用
- 路由與導航規則對你的頁面跳轉很重要
- 安全性規則確保使用者資料不會外洩

---

## 3. project-architecture.mdc（專案架構規則）

### 🎯 用處（大白話）
這個規則告訴 AI 整個專案應該怎麼組織，就像蓋房子要有藍圖一樣：
- **目錄結構**：檔案要放在對的地方，不能亂放，要找的時候才找得到
- **組件架構**：每個組件（像是登入表單、帳戶列表）要獨立，可以重複使用
- **狀態管理**：使用者的登入狀態、資料要怎麼管理，不能到處亂存
- **API 整合**：跟 Supabase 的溝通要統一處理，不能每個地方都寫一遍
- **安全性架構**：登入驗證、權限控制要怎麼設計
- **部署架構**：怎麼用 GitHub Pages 部署上線

### 💡 點評
**優點：**
- 提供了清晰的專案結構範本，讓專案不會亂成一團
- 特別強調安全性架構，這對你的管理員權限功能很重要
- 考慮到擴展性，未來要加功能時不會太困難

**特別適用於你的專案：**
- 目錄組織規則讓你的專案結構清晰
- 安全性架構規則對你的管理員/一般使用者權限區分很重要
- 部署架構規則直接對應你要用 GitHub Pages 的需求

---

## 🔧 如何運作

這些規則已經設定為 `alwaysApply`，這意味著：
- ✅ 每次與 AI 對話時，這些規則會自動生效
- ✅ AI 會根據這些規則來生成程式碼和建議
- ✅ 不需要每次都提醒 AI 要遵守什麼規則
- ✅ 確保整個開發過程的一致性

## 📝 使用建議

1. **開發時**：AI 會自動遵循這些規則，幫你寫出符合標準的程式碼
2. **遇到問題時**：可以參考這些規則文件，了解為什麼 AI 會這樣建議
3. **需要調整時**：如果發現規則不適合，可以修改 `.mdc` 文件
4. **團隊協作時**：這些規則確保所有成員都遵循相同的標準

---

## 🎯 總結

這三個規則文件形成了一個完整的開發指南：
- **fundations.mdc**：確保程式碼品質和安全性
- **front-developer.mdc**：確保前端開發符合最佳實踐
- **project-architecture.mdc**：確保專案結構清晰可維護

對於你的帳戶管理系統專案來說，這些規則特別重要，因為：
1. 涉及使用者資料，安全性必須嚴格
2. 需要區分管理員和一般使用者，權限架構要清楚
3. 要部署到 GitHub Pages，結構要清晰
4. 使用 Supabase，API 整合要統一

有了這些規則，AI 就能更準確地理解你的需求，並生成符合你專案標準的程式碼！
